## 🚀 快速开始

### 环境准备

1. 支持 **Windows / macOS** 双平台，推荐使用 **MuMu 模拟器 12**（Windows）或 Android 模拟器（macOS）
2. 建议使用**横屏分辨率**，脚本会自动适配
3. MuMu 模拟器默认 ADB 地址为 `127.0.0.1:16384`（多开请到模拟器设置内查看对应端口）
4. 自备加速器，保证网络通畅
5. 游戏语言**必须设置为简体中文**

### MuMu 模拟器注意事项

- 优先连接 `127.0.0.1:16384` / `127.0.0.1:16416` 等端口，尽量不要连接 `127.0.0.1:5555` / `emulator-5554`
- 在设备设置中**关闭**"网络桥接模式"
- **关闭**"后台挂机时保活运行"选项
- 如 nemu_ipc 方案无法启用，手动指定 MuMu 安装路径（例如 `D:\Program Files\MuMuPlayer`，不要指定到 `nx_main` 子文件夹）
- 如遇未知问题，尝试切换模拟器渲染模式为 **DirectX**

### 启动脚本

1. 运行 `WOA_AutoBot.exe`
2. 点击 **🔄 刷新设备** 选择模拟器
3. 点击 **▶ 启动**，脚本开始自动处理任务

## ⚠️ 使用须知

1. **请勿与脚本同时手动操作**——手动操作前请先停止脚本运行
2. 脚本使用双击空白处的方式关闭窗口（默认右上角附近），如误触飞机请调整挂机视角或将视角拉到最近
3. 机位分配只会点第一个，如需筛选机位停机类型请手动设置，且与时刻表功能不兼容（需重置时刻表）
4. 任何情况下脚本**没有**滑动右侧任务列表的能力

## 🎛️ 设置详解

### 截图与触控方案

| 方案 | 速度 | 说明 |
|------|------|------|
| uiautomator2 + ADB | ⚡ 最快 | 推荐方案，触控用 uiautomator2，截图用 ADB |
| ADB + ADB | 🐢 较慢 | 兼容性最好，全 ADB 方案 |
| nemu_ipc + ADB | ⚡ 快 | MuMu 专属高速截图，需指定 MuMu 路径 |

> ⚠️ 使用高速方案时出错会增多，**不建议**关闭"跳过二次校验"和"跳过地勤分配验证"。

### 筛选模式

- **仅待处理**：只显示黄色感叹号待处理任务（默认，脚本会自动检测并调整筛选状态，支持 4 按钮识图 + 模板匹配交叉验证）
- **仅停机位**：只显示停机位任务（塔台全开时生效）

### 不起飞模式

- 开启后只处理进港和停机位任务，不处理起飞
- 自动在"待降落"和"停机坪"之间轮切
- 支持定时自动小退换机场

### 塔台延时

- 使用前请确保**已开启塔台**并设置好延时按钮界面（脚本不会主动调整界面布局）
- 塔台全开时自动一键**全部续费**；部分开启时单独续费每个控制器
- 红灯最高优先级，检测到立即处理
- 支持二次确认弹窗自动点击

### 防卡死

- 连续超时自动触发紧急恢复
- 可在设置中调整触发阈值（3-20）
- 多次自修复失败后自动停机保护

## ❓ 常见问题

**Q: 截图失败怎么办？**
A: 尝试切换截图方案：设置 → 截图方案 → 选择 ADB / nemu_ipc / DroidCast

**Q: 触控不生效？**
A: 尝试切换触控方案：设置 → 触控方案 → 选择 ADB / uiautomator2

**Q: 脚本频繁关闭窗口？**
A: 检查是否在正确的游戏界面（机场主界面），确保任务列表可见

**Q: nemu_ipc 方案无法启用？**
A: 手动指定 MuMu 安装路径（设置 → MuMu 路径），路径到 `MuMuPlayer` 级别即可

**Q: 模拟器连接不上？**
A: 先尝试手动指定 ADB 路径；检查是否连接了正确的端口（优先 16384/16416）

**Q: 脚本收费吗？**

## ⚠️ 已知问题和缺陷

1. 脚本支持多开但测试不充分，多开可能存在未知问题。运行中开启/关闭第二个脚本或类似软件（如 ALAS）会导致运行中断，请停止后再重新运行
2. 脚本可能被杀毒软件误杀，如遇此类问题请关闭杀毒软件或添加信任
3. 处理"需要维护"的飞机时暂无法应对绿币不足，请根据机队规模预留充足绿币
4. 任何情况下脚本都没有滑动右侧任务列表的能力
