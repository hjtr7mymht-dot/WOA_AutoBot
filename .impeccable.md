## Design Context

### Users
游戏自动化工具的桌面用户，在 Windows/Mac 上长期挂机运行，偶尔查看仪表盘监控数据。核心需求：高效、直观、长时间使用不疲劳。

### Brand Personality
**VS Code × 游戏启动器** — 专业开发工具感 + 克制游戏氛围。
三个关键词：**克制、清晰、工具感**
- 克制：无蓝色调底色，中性深灰打底，颜色仅 10% accent
- 清晰：WCAG AA (≥4.5:1) 正文对比度，信息层级通过灰度递进
- 工具感：像 VS Code 一样的专业桌面工具，不做"主题皮肤"

### Aesthetic Direction
**方向**：Professional Game Tool — VS Code Dark+ 中性灰 × 克制冷色点缀
- 主色调：VS Code 蓝 (#007acc)，仅用于 accent / CTA / 焦点
- 底色系统：纯中性灰度 18%–38% (暗色) / 94%–100% (浅色)，无蓝调
- 语义色：VS Code 风格 (青绿 success、金黄 warning、柔和红 danger)
- 色彩比例：80% 中性、15% 次要元素、5% accent（比 60-30-10 更极端克制）
- 反参考：霓虹色、发光边框、蓝黑底色、飞行仪表 HUD、任何"主题化"装饰

### Design Principles
1. **中性灰底色**：暗色 #1e1e1e 起 (VS Code editor bg)，浅色 #ffffff 白底，绝不用蓝/紫/绿调灰色
2. **Accent 极克制**：#007acc 仅用于关键交互元素，绝不做大面积蓝色面板
3. **去装饰化**：无渐变、无阴影、无发光、无线条装饰——纯扁平面板+1px 边框
4. **Token 全覆盖**：所有颜色通过 `self._clr` 字典引用，零硬编码 hex
5. **10:1 文字对比度**：暗色 #cccccc on #1e1e1e，浅色 #1e1e1e on #ffffff

### Technical Notes
- 框架：ttkbootstrap (darkly/flatly 主题) + 原生 tkinter
- 色彩 Token 名：`self._clr` dict，键名含 bg、surface、border、primary、text 等
- 日志标签刷新：`redirector.refresh_tags(c)` 在 `_apply_theme_colors` 末尾调用
- 面板底色统一：卡片/面板用 `c["surface"]` 而非专用蓝色
